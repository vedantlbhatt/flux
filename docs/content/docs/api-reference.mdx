## Endpoints

| Method | Path | Purpose |
|---|---|---|
| `GET` | `/health` | Service readiness by provider key availability. |
| `GET` | `/search` | Live Tavily retrieval + Cohere reranking. |
| `GET` | `/answer` | Synthesized answer with citations from live sources. |
| `GET` | `/contents` | Extract readable content from specific URLs. |
| `POST` | `/conversations` | Create a new conversation context. |
| `GET` | `/conversations` | List conversations with pagination. |
| `GET` | `/conversations/{id}` | Retrieve full conversation history. |
| `POST` | `/conversations/{id}/messages` | Add context-aware message and receive answer. |
| `DELETE` | `/conversations/{id}` | Delete a conversation. |

## Search: `GET /search`

### Query Parameters

- `q` (required): natural language query, max 500 chars
- `limit` (optional): default `10`, range `1..20`
- `topic` (optional): `news | general`
- `days` (optional): positive integer

### Response Highlights

- `results[]` includes title, URL, snippet, score, and rank.
- `reranked` indicates if semantic reranking was applied.
- `total` indicates result count returned.

## Answer: `GET /answer`

### Query Parameters

- `q` (required)
- `topic` (optional)
- `days` (optional)

### Response Highlights

- `answer`: concise natural language synthesis
- `citations[]`: source references used for synthesis
- `model`: synthesis model identifier

## Contents: `GET /contents`

### Query Parameters

- `urls` (required): comma-separated list of URLs, max 10

### Response Highlights

Each item includes:

- `url`
- `title`
- `content`
- `word_count`
- `success`

## Error Model

All API errors follow:

```json
{
  "error": "Human readable message",
  "code": "MACHINE_READABLE_CODE"
}
```

Typical codes include:

- `MISSING_QUERY`
- `QUERY_TOO_LONG`
- `INVALID_TOPIC`
- `INVALID_DAYS`
- `MISSING_URLS`
- `TOO_MANY_URLS`
- `CONVERSATION_NOT_FOUND`
- `NO_RESULTS`
- `TAVILY_ERROR`
- `ANSWER_FAILED`
- `INTERNAL`
